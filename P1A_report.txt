In terms of macros we have START_READ(), END_READ(), START_WRITE(), END_WRITE(). There are basically placeholders that get replaced by actual function calls such as startRead(), doneRead(), startWrite(), doneWrite() at compile time if P1_RWLOCK is passed, which is passed for 1a. The compiler will replace START_WRITE() with locks[hash].startWrite(), END_WRITE() with locks[hash].doneWrite() and so on at compile time. We built our functions using Pthreads condition variables and mutex locks as describe in the lecture slides. The RWLock class keeps track of 3 counters, readers: where there can be multiple readers, writers: only 0 or 1 active writers and lastly writers_waiting: used to keep track of how many writers are waiting. There are all protected by a single mutex. We also added two condition variables, readers_proceed and writers_proceed. Our implementation guarantees that once a writer arrives, no new readers can come in and prevent writer starvation. All four reading and writing functions begin with the mutex lock and end with mutex unlock. For the startRead() function, it waits if a write is active or waiting, with writer getting priority, otherwise increments the reader count and proceeds. doneRead() function implements reader finishes by decrementing the reader count and if it was the last reader and a writer is waiting, it wakes up one writer. startWrite() function implements when a writer tries to enter by incrementing waiting_writers, waits if any readers or another writer is active and once its safe, marks itself as the active writer. doneWrite() implements when the writer finishes, clears the writer flag, if the writers are waiting then wake up one writer otherwise wake up all readers. RWLock() sets all 3 counters to 0 and initializes the mutex and condition variables. ~RWLock() destroys the mutex and condition variables to prevent memory leaks.
